<blockquote>
<p>최근에 면접을 하나 더 봤는데요. 그때 받았던 질문들을 정리해보겠습니다.</p>
</blockquote>
<p>일반 면접과 마찬가지로 처음에는 본인 소개 및 회사에 대해 어느정도 아는지 물어보셨다.</p>
<p>본인이 관심이 있다면 당연히 대답할 수 있어야 한다. 본격적으로 질문을 적어보도록 하겠다.</p>
<h2 id="1-drm">1. DRM</h2>
<blockquote>
<p><strong>Digital Rights Management</strong>의 약자로서, 디지털 콘텐츠의 사용 권한을 관리하는 기술이다. </p>
</blockquote>
<p>저작권 보호를 통해 콘텐츠 제작자의 권리를 보호하고, 콘텐츠의 불법 복제 및 배포를 어렵게 만들어 산업의 성장을 지원한다.</p>
<p>일반적으로 외부 서비스나 소트프웨어를 통해 구현된다고 한다.</p>
<h2 id="2-슬라이딩-윈도우">2. 슬라이딩 윈도우</h2>
<blockquote>
<p>이 부분이 확실하진 않은데 들었던 것 같아 정리한다.</p>
</blockquote>
<p>배열이나 리스트의 <strong>일정 범위의 요소를 연속적으로</strong> 이동시키면서 문제를 해결하는 방법이다. <strong>최대값, 최소값, 합계</strong> 등을 빠르게 계산할 때 유용하다.</p>
<p>윈도우 설정 -&gt; 윈도우 이동 -&gt; 연산 수행 -&gt; 최적의 결과 도출이 기본적인 순서이다.</p>

<p>3개의 파트를 맡아 진행했었는데 그 중 정렬 파트에 대해 간략하게 설명드렸다. TimSort를 사용한다고 했는데 이 부분에 대해 추가 질문을 하셨다.</p>

<p>나는 지금 당장은 경험이 부족해서 팔로워라고 생각하고 후에는 리더를 해볼 것이다라는 느낌으로 말했다.</p>
<h2 id="7-리더가-되면-어떻게-이끌-것인가">7. 리더가 되면 어떻게 이끌 것인가</h2>
<p>모두가 성장할 수 있는 환경을 만들고, 아는 부분에 대해선 공유하게끔 할 것이다.. 라고 했다. </p>
<h2 id="8-본인의-장단점강점-약점">8. 본인의 장단점(강점, 약점)</h2>
<p>새로운 것을 좋아하고 배움에 있어서 겸손하다고 말씀드렸다. 약점은 인정받고자 하는 욕구라고 했다.</p>
<h2 id="9-해당-약점을-약점이라고-생각하는-이유">9. 해당 약점을 약점이라고 생각하는 이유</h2>
<p>아직은 그런 적이 없지만 극단적으로 생각해보면 인정을 받기 위해 수단과 방법을 가리지 않을 수도 있겠다 라는 생각에 그렇게 말씀드렸다.</p>
<p>이를 방지하기 위해 관련된 사례들을 파악하고 주변인에게 도움을 구하겠다 라고 했다.</p>
<h2 id="10-tcp-udp-차이점">10. TCP, UDP 차이점</h2>
<blockquote>
<p>인터넷 상에서 데이터를 전송하기 위해 사용되는 두 가지 주요 프로토콜이다.</p>
</blockquote>
<h3 id="tcp">TCP</h3>
<p><strong>3-way handshake</strong>를 사용하며 <strong>데이터의 오류를 검사</strong>하고 오류 없이 데이터를 전송한다. 이메일, 웹 페이지 로딩, 파일 전송 등에 사용한다.</p>
<h3 id="udp">UDP</h3>
<p>3-way handshake 같은 연결을 사전에 설정하지 않는다. 또한, <strong>손실이 발생해도 재전송을 요청하지 않는다.</strong> 실시간 스트리밍, 온라인 게임 등에 사용한다.</p>
<h2 id="11-rest-api-원칙">11. REST API 원칙</h2>
<blockquote>
<p>Representational State Transfer의 약자로, 웹 표준을 활용하여 시스템 간의 상호 운용성을 제공하는 방법이다. </p>
</blockquote>
<p><strong>자원(Resource)</strong>의 <strong>표현(Representation)</strong>에 의한 상태 전달을 의미하며, 웹의 기본적인 기술과 프로토콜을 사용한다.</p>
<p>이를 정의하기 위해 몇가지 핵심 원칙을 따라야 한다.</p>
<h3 id="1-명사를-사용하라">1. 명사를 사용하라</h3>
<p>항상 리소스(객체)가 어느 범주에 해당하는지 확인하고, 그 범주에 맞는** 네이밍 컨벤션<strong>을 **일관되게 사용</strong>해야 한다.</p>
<p>무조건 동사를 쓰지 말라는 것은 아니다.</p>
<h3 id="2-일관성이-핵심이다">2. 일관성이 핵심이다</h3>
<p>가독성과 지속성을 극대화 하기 위해 <strong>'/'</strong> 나 <strong>'-'</strong>를 사용한다. </p>
<p>또한, <strong>소문자</strong>를 사용하고, <strong>파일 확장자</strong>를 사용하지 않는다.</p>
<p>마지막 문자로만 사용하지 말자.</p>
<h3 id="3-crud-함수명을-사용하지-마라">3. CRUD 함수명을 사용하지 마라</h3>
<p><strong>URI</strong>는 어떤 동작이 수행되는 지를 가르키는 것이 아니라, 리소스를 가르킨다.</p>
<p>리소스에 대한 작업은 <strong>HTTP Method</strong>를 이용한다.</p>
<h3 id="4-필터를-위해-쿼리-파라미터를-사용해라">4. 필터를 위해 쿼리 파라미터를 사용해라</h3>
<p>리소스에 대한 정렬, 필터링, 페이징은 신규 API를 생성하지 않고 쿼리 파라미터를 활용한다. </p>
<h2 id="12-왜-동사를-사용하면-안될까요">12. 왜 동사를 사용하면 안될까요?</h2>
<p>다양한 이유가 있지만 동사를 사용하면 같은 리소스에 대한 동작도 이름이 매우 달라진다. </p>
<p>반면에, 명사를 사용하면 api를 규칙적이고 깔끔하게 관리할 수 있다. </p>
<pre><code class="language-python">GET  /getUsers         ---&gt; GET  /users
POST /createNewUser    ---&gt; POST /users
GET  /getUserById/12   ---&gt; GET  /users/12</code></pre>
<p>또한, 프로퍼티를 표현하기에 적합하다.</p>
<pre><code class="language-python">GET  /getAccountsOfUserByUserId/123   
---&gt; GET  /users/123/accounts</code></pre>
<h2 id="13-http-상태코드">13. HTTP 상태코드</h2>
<blockquote>
<p>HTTP 요청에 대한 상태를 나타내는 코드이다.</p>
</blockquote>
<ul>
<li><strong>1xx(정보)</strong>: 요청을 받았으며 프로세스를 계속 진행</li>
<li><strong>2xx(성공)</strong>: 요청을 성공적으로 받았으며 인식했고 수용했음<ul>
<li><strong>200 OK</strong> -&gt; 요청이 성공적으로 실행됨</li>
<li><strong>201 Created</strong> -&gt; 요청이 성공적이며 그 결과 새로우 리소스가 생성됨</li>
</ul>
</li>
<li><strong>3xx(리다이렉션)</strong>: 요청 완료를 위해 추가 작업 조치가 필요함</li>
<li><strong>4xx(클라이언트 오류)</strong>: 요청의 문법이 잘못되었거나 요청을 처리할 수 없음<ul>
<li><strong>400 Bad Request</strong>: 잘못된 문법으로 인해 서버가 요청을 거절함</li>
<li><strong>403 Forbidden</strong>: 클라이언트가 콘텐츠에 접근할 권한이 없음</li>
</ul>
</li>
<li><strong>5xx(서버 오류)</strong>: 서버의 오류 혹은 프록시 서버나 게이트웨이 같은 서버의 보조 구성 요소에 문제가 생겼음</li>
</ul>

<p>답변을 제대로 못했는데 면접관님이 시간 부족했으니 이해한다..? 라는 느낌으로 넘어가 주셨다.</p>

<h3 id="3-댓글-신고-필터링이-의미가-없음">3. 댓글 신고 필터링이 의미가 없음</h3>

<p>content 필드를 Serializer 필드에 추가하고 get_content 메서드에서 댓글 작성자인 경우, 댓글 내용을 직접 반환해야 한다.</p>
<h3 id="4-댓글-신고-카운트-상승을-1씩-더하게-구현한-이유">4. 댓글 신고 카운트 상승을 1씩 더하게 구현한 이유</h3>
<p>단순하게 생각하여 누르면 1씩 증가하게 구현했다. 그렇지만 여기에는 문제가 존재한다.</p>
<h4 id="동시성-문제">동시성 문제</h4>

<p>이를 해결하기 위해 트랜잭션, F() 표현식으로 처리할수도 있다.</p>

<p>추가로, 시간이 부족하셨냐.. 라고 물어보셨는데 그에 대해서 솔직하게 부족했다고 말씀드렸다. 그러나 점심 시간을 좀 줄여서 진행했으면 완성했을 것이라고 했다.</p>
